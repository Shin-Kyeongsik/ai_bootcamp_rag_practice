# AI 교육 부트캠프 안내 — RAG / LLM Wiki / RAG+Wiki 실습

같은 원본 문서(`bootcamp_docs/`)를 가지고 **세 가지 방식**을 각각 "어떻게 구성하는지" 단계별로 따라가는 실습 자료입니다.
대상 부트캠프는 가상의 **AI Agent 실무 부트캠프 2026**(2026년 8월 3일 월 ~ 8월 7일 금)입니다.

## 폴더 구조

| 폴더 | 내용 |
|------|------|
| `bootcamp_docs/` | 공통 소스 — 가상 부트캠프 원본 안내 문서 9개 (검색/재구성 대상) |
| `practice_prompts/` | 실습용 질문 예시 |
| `01_rag/` | **RAG** — 원본 문서를 그대로 청킹·검색해서 답변하는 과정 |
| `02_llm_wiki/` | **LLM Wiki** — LLM이 원본을 위키로 재구성하고, 그 위키로 답변하는 과정 |
| `03_rag_llm_wiki/` | **RAG + LLM Wiki** — 재구성한 위키에 RAG(검색)를 붙이는 과정 |

## 권장 학습 순서

1. **`01_rag/`** — RAG의 기본 파이프라인(로드 → 청킹 → 임베딩 → 검색 → 생성)을 스텝별로 익힌다.
2. **`02_llm_wiki/`** — 검색 대신 "LLM이 지식을 미리 정제(위키화)"하는 접근을 익힌다.
3. **`03_rag_llm_wiki/`** — 02에서 만든 위키를 검색 대상으로 삼아 RAG와 결합한다. (02를 먼저 실행)

각 폴더는 `step1_*.py → step2_*.py → …` 순서로 실행하며, 실행할 때마다 중간 산출물(청크·임베딩·검색 결과·생성된 위키)을 눈으로 확인할 수 있습니다. 자세한 실행법은 각 폴더의 `README.md`를 참고하세요.

## 통합 데모 앱 (세 방식 비교)

세 방식을 **사이드바에서 골라 전환**하며 같은 질문으로 비교하는 웹 앱입니다.

```bash
streamlit run app.py
```

- 사이드바에서 `① RAG(원본) / ② LLM Wiki(위키 전체) / ③ RAG+Wiki(위키 검색)` 선택
- 각 방식에 맞는 답변·근거·검색된 조각을 표시
- **사전 조건**: 각 방식이 쓰는 산출물이 먼저 있어야 합니다.
  - ① → `01_rag/step2_chunk.py`, `step3_embed_store.py`
  - ② → `02_llm_wiki/step2_build_wiki.py`
  - ③ → ② 실행 후 `03_rag_llm_wiki/step2_chunk_wiki.py`, `step3_embed_store.py`

## 사전 준비

```bash
pip install -r requirements.txt
```

OpenAI API 키는 프로젝트 루트의 **`api_key.txt`** 에 저장돼 있으며, 코드가 실행 시 자동으로 불러옵니다.
따라서 수강생은 **`pip install` 후 바로 스텝을 실행**하면 됩니다(별도 키 설정 불필요).

> 직접 다른 키를 쓰고 싶으면 `export OPENAI_API_KEY=sk-...` 로 환경변수를 지정하면 됩니다(환경변수가 있으면 그 값을 우선 사용).
>
> ⚠️ `api_key.txt`에는 실제 키가 들어 있습니다. **인트라넷 공유 전용**이며, 외부 저장소/깃 원격에 올리지 마세요. 키에는 사용량 상한(Usage limit)을 걸어두길 권장합니다.

## 세 방식의 핵심 차이

| 방식 | 지식을 언제 정리하나 | 검색 사용 | 답변 근거 |
|------|----------------------|-----------|-----------|
| RAG | 정리 안 함(원본 그대로) | ✅ 질의 시점에 원본 조각 검색 | 원본 청크 |
| LLM Wiki | **미리** LLM이 위키로 정제 | ❌ (위키 전체를 컨텍스트에 투입) | 위키 전체 |
| RAG + Wiki | **미리** 위키로 정제 | ✅ 위키 조각 검색 | 위키 청크 |
