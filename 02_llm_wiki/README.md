# 02_llm_wiki — LLM으로 위키를 만들고 활용하기

원본 문서를 **LLM이 미리 위키로 재구성**하고, 그 위키를 통째로 근거로 답변하는 방식을 단계별로 만듭니다.
RAG처럼 질의 시점에 '검색'하지 않고, 지식을 **먼저 정제**해 두는 것이 핵심 차이입니다.

## 파이프라인 한눈에

```
문서 로드 → LLM이 위키로 재구성 → (확인) → 위키 전체로 답변
step1       step2                  step3     step4
```

## 실행 순서

```bash
# API 키는 ../api_key.txt 에서 자동으로 불러옵니다 (별도 설정 불필요)

python step1_load.py                       # ① 원본 + 주제 매핑 확인
python step2_build_wiki.py                 # ② LLM이 위키 생성 → wiki/*.md
python step3_inspect_wiki.py               # ③ 생성된 위키 확인 (선택)
python step4_answer_from_wiki.py "수료 기준은?"   # ④ 위키 전체로 답변 (검색 없음)
```

## 각 스텝이 보여주는 것

| 스텝 | 하는 일 | 눈으로 확인하는 산출물 |
|------|---------|------------------------|
| step1_load | 원본 + 주제 매핑 | 원본 목록, 위키 페이지 매핑 |
| step2_build_wiki | LLM이 위키 재구성 | `wiki/*.md` (요약·통합된 위키) |
| step3_inspect_wiki | 위키 살펴보기 | 페이지 미리보기, 원본 대비 압축률 |
| step4_answer_from_wiki | 위키로 답변 | 답변 + 사용한 위키 페이지 |

## RAG(01) vs LLM Wiki(02)

| | RAG (01_rag) | LLM Wiki (02_llm_wiki) |
|---|---|---|
| 지식 정리 시점 | 안 함(원본 그대로) | **미리** LLM이 위키로 정제 |
| 질의 시 검색 | ✅ 관련 청크만 검색 | ❌ 위키 전체를 컨텍스트에 투입 |
| 근거 | 원본 청크 | 위키 페이지 |
| 장점 | 원본 그대로, 대규모에 유리 | 정리된 지식, 문맥 일관성 |
| 한계 | 원본이 지저분하면 검색 품질 저하 | 위키 생성 비용, 대규모 문서는 컨텍스트 초과 |

> 참고: 이 실습의 문서는 작아서 위키 전체가 컨텍스트에 들어갑니다. 문서가 많아지면
> 위키에도 검색이 필요해지는데, 그것이 다음 모듈 **03_rag_llm_wiki** 입니다.

## 산출물

- `wiki/*.md` 는 step2가 생성하는 산출물입니다. 03_rag_llm_wiki가 이 위키를 입력으로 사용합니다.
