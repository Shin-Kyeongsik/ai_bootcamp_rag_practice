# RAG 질문 예시

이 질문들은 일반 LLM이 정확히 알 수 없는 부트캠프 고유 정보를 묻는다.  
반드시 `bootcamp_docs` 원본 문서를 검색해서 답변해야 한다.

---

## 1. 기본 질문

주로 하나의 문서만 확인하면 답할 수 있는 단일 정보 질문이다.

1. 등록은 몇 시부터 어디서 하나요?
2. 교육장은 어디인가요?
3. Docker 설치가 안 되어 있으면 언제 도움을 받을 수 있나요?
4. 일일 과제는 언제까지 제출해야 하나요?
5. 최종 과제는 언제까지 제출해야 하나요?
6. 수료 기준은 무엇인가요?
7. 멘토링은 매일 몇 시에 진행되나요?
8. 최종 발표는 언제 진행되나요?
9. 주차권은 어디서 받을 수 있나요?
10. Python 권장 버전은 무엇인가요?

---

## 2. 여러 문서를 함께 봐야 하는 질문

여러 문서에 흩어진 정보를 조합해야 정확히 답할 수 있는 복합 질문이다.

1. Docker 설치가 안 된 참가자는 언제, 어디서, 어떤 방식으로 도움을 받아야 하나요?
2. 수료를 위해 과제와 발표 측면에서 놓치면 안 되는 것은 무엇인가요?
3. 첫날 참가자가 등록부터 첫 세션까지 따라야 할 순서를 알려주세요.
4. 최종 발표를 준비하려면 5일차 14:30 전까지 무엇을 해야 하나요?
5. 멘토에게 질문하기 전에 참가자가 먼저 준비해야 할 것은 무엇인가요?
6. 개인 노트북을 가져오지 않으면 어떤 문제가 생기나요?
7. 과제를 제출했지만 마감 이후에 push하면 어떻게 처리되나요?
8. 지각이 반복되면 수료 기준에 어떤 영향을 주나요?
9. 2일차 오전에는 어떤 환경 점검과 지원이 있나요?
10. 최종 프로젝트 발표에서 팀이 반드시 보여줘야 하는 것은 무엇인가요?

---

## 3. 일반 LLM과 비교하기 좋은 질문

아래 질문은 일반 LLM에게 먼저 물어본 뒤, RAG 시스템에 다시 물어보면 차이를 비교하기 좋다.

1. AI Agent 실무 부트캠프 2026의 등록 장소는 어디인가요?
2. 이 부트캠프의 Docker 설치 지원 시간은 언제인가요?
3. 이 부트캠프의 수료 기준은 무엇인가요?
4. 최종 과제는 언제까지 제출해야 하나요?
5. 멘토링은 어디에서 진행되나요?

일반 LLM은 이 정보를 알 수 없으므로 추측하거나 일반적인 답변을 할 가능성이 높다.  
RAG 시스템은 `bootcamp_docs` 문서를 근거로 정확히 답해야 한다.

## 5. 답변 예시

**질문:**  
Docker 설치가 안 되어 있으면 언제 도움을 받을 수 있나요?

**답변:**  
Docker 설치 지원은 2일차인 2026년 8월 4일 화요일 10:30~11:00에 진행됩니다. 장소는 A동 3층 Lab 301 앞 멘토 데스크입니다.

**근거 문서:**

- `bootcamp_docs/04_실습_환경_설치_가이드.md`
- `bootcamp_docs/02_전체_일정표.md`

**추가로 확인하면 좋은 내용:**

- Python 권장 버전은 3.11입니다.
- 설치 문제가 있으면 에러 메시지를 캡처한 뒤 멘토링 시간(매일 17:00~18:00)에 질문할 수 있습니다.
